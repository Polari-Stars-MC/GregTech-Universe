# 1.21.1 NeoForge 工业文明整合包总方案

## 1. 项目定位

这是一个以 **Create + GregTech CEu Modern + 自研星际模组 + 自研物理模组** 为骨架的超长线整合包。

核心目标不是“堆科技模组”，而是让玩家按接近真实工业史的方式，亲历：

- 石器与青铜
- 水力与风力机械
- 蒸汽革命
- 石油与内燃机
- 电气化
- 航空与火箭
- 轨道工业
- 行星开发
- 戴森云与恒星工程

平台锁定为 **Minecraft 1.21.1 + NeoForge**。

当前清单为 **296 项**：

- 外部模组 / 依赖：266
- 自研模块：30

其中：

- **星际内容全部自研**
- **物理内容全部自研**
- Create 在前中期是绝对主角
- GTM 负责 14 电压阶段主线
- 后期以自研航天系统承接 LuV 之后的内容

## 15. 每阶段的具体玩法落点

### 15.1 0-100 小时：现实工业史 1:1 前史

1. `0-10h`
   - 徒手生存、燧石、黏土、炭窑。
   - Create 配方全部偏手工化，禁止玩家开局直跳电力。
2. `10-25h`
   - 用 Create 做第一条机械传动线。
   - 前期矿石处理由粉碎轮、磨盘、压印机承担。
3. `25-40h`
   - 建立机械工坊和早期冶炼体系。
   - 通过任务线引导玩家理解扭矩、转速、应力。
4. `40-70h`
   - 开启蒸汽锅炉、蒸汽机、蒸汽矿物线。
   - 铁路开始承担跨区运输。
5. `70-100h`
   - 找石油、搭蒸馏塔、做汽油和柴油。
   - Create 不再是唯一动力，但仍是物流骨架。

### 15.2 100-280 小时：电气时代到火箭先驱

1. `LV`
   - 任务强制玩家经历第一次真正的电网搭建。
   - 铝与铜线生产是第一道工业门槛。
2. `MV`
   - 钢铁厂、轧机、结构钢流水线形成。
   - 自研物理开始作用在机库门、塔吊、简易飞机上。
3. `HV`
   - 真空管、无线电、雷达、远程制导。
   - 自研星际模组只开放“探空火箭”和试验台，不开放完整行星飞行。

### 15.3 280-520 小时：轨道工业与月火时代

1. `EV`
   - 人造卫星、通信链路、轨道发射任务。
   - 发射流程必须包含燃料、倒计时、窗口校验。
2. `IV`
   - 登月、返回、轨道交会、空间站建设。
   - 空间站是后续 LuV 以后很多工艺的环境需求。
3. `LuV`
   - AE2 变成真正的中枢。
   - 玩家开始把轨道基地当作工业延伸，而不是“旅游景点”。

### 15.4 520-1000 小时：行星开发到恒星工程

1. `ZPM`
   - 火星常驻点、小行星拖船、轨道燃料补给。
2. `UV`
   - 木星冰卫星勘探，核热火箭缩短运输时间。
3. `UHV`
   - 聚变反应堆上线，恒星际探测器发射。
4. `UEV / UIV`
   - 戴森云、恒星调光、行星改造。
5. `UMV / UXV / MAX`
   - 戴森球完工，进入终局成就链。

## 16. 自研星际模组设计边界

### 16.1 必须自己写的原因

- 现有 1.21.1 NeoForge 航天生态很难同时满足 **真实工业史递进 + GTM 电压门槛 + 轨道工业化 + 戴森云终局**
- 大多数现成航天模组要么太“点按钮式”，要么科技树过短
- 你这个整合包要的是 **工业文明史诗感**，不是单次上天

### 16.2 自研星际模组的四层结构

1. **发射层**
   - 火箭分级
   - 发动机推力曲线
   - 燃料温度与储箱
2. **轨道层**
   - 近地轨道
   - 转移轨道
   - 会合 / 对接
3. **工业层**
   - 轨道装配
   - 太空冶炼
   - 轨道物流
4. **终局层**
   - 戴森云
   - 戴森球
   - 恒星工程

### 16.3 和 GTM 的绑定方式

- LV-HV：只开放实验台、燃料、简易火箭结构件
- EV-IV：开放载人航天、卫星、空间站
- LuV-ZPM：开放行星基地、轨道工业、深空探测
- UV-UHV：开放核热与聚变推进
- UEV 以上：开放戴森工程

## 17. 自研物理模组设计边界

### 17.1 物理模组不要做成“万物模拟器”

建议只做三类和玩法强相关的物理：

- **结构物理**
  - 火箭发射架
  - 高塔
  - 大型吊车
  - 轨道桥
- **流体压强**
  - 蒸汽锅炉
  - 高压燃料管线
  - 液氧 / 液氢储运
- **载具物理**
  - 飞机
  - 火箭
  - 列车
  - 大型工程车辆

### 17.2 不建议做的内容

- 全世界所有方块实时刚体化
- 全流体 CFD 级模拟
- 每个 Create 零件都做连续物理

原因很简单：会炸性能，也会把整合包重心从“工业发展”带偏成“物理沙盒”。

## 18. 配方与门槛设计原则

### 18.1 前期必须硬控现实顺序

- 石头 -> 铜 / 锡 -> 青铜 -> 铁 -> 钢 -> 铝 -> 钛 -> 超合金
- 不允许玩家靠其他模组跳材料年代
- 所有“现代材料”必须被 GTM 和任务线锁死

### 18.2 Create 与 GTM 的职责分工

- `0-100h`：Create 是主系统，GTM 只是远方目标
- `100-400h`：Create 负责物流与机械构造，GTM 负责主科技
- `400h+`：AE2 接管高密度物流，Create 回归为大型工程骨骼

### 18.3 支线模组都必须服从主线

以下模组只能作为支线或工具，不能替代 GTM：

- Extreme Reactors
- RFTools Power
- QuarryPlus
- Building Gadgets 2
- Time in a Bottle

做法：

- 用 KubeJS 重写核心配方
- 关键机器加阶段标签
- 通过任务和战利品解锁蓝图

## 19. 整合包制作落地流程

### 阶段 A：最小可运行原型

1. 先只装：`NeoForge + GTM + Create + JEI/EMI + Jade + KubeJS + FTB Quests + Embeddium + ModernFix`
2. 跑通客户端和服务器
3. 确定 Java 21、内存、启动参数

### 阶段 B：前 100 小时原型

1. 写 KubeJS 配方，强制机械纪元和蒸汽纪元
2. 做 0-100 小时任务线
3. 先不接航天，只验证前期“历史感”是否成立

### 阶段 C：GTM 电压主线成型

1. 把 LV -> IV 的机器、材料、矿处理全做完
2. 做阶段成就与演出事件
3. 跑一次单人生存压测到 IV

### 阶段 D：自研物理模组接入

1. 只先接 `结构应力 + 流体压强`
2. 不要一开始就做复杂刚体
3. 先让锅炉、塔架、燃料管线能稳定工作

### 阶段 E：自研星际模组第一版

1. 先做探空火箭
2. 再做卫星和近地轨道
3. 再做登月与空间站

### 阶段 F：中后期内容闭环

1. 接入 AE2 主网
2. 做火星、小行星、木星任务
3. 把 LuV 以后的材料需求与航天环境绑定

### 阶段 G：终局与奇观

1. 做戴森云 / 戴森球工程
2. 做终局成就、奇观动画、终章任务线
3. 做服务器耐久压测

## 20. 推荐的开发优先级

| 优先级 | 先做什么 | 原因 |
|---|---|---|
| P0 | GTM + Create + KubeJS + FTB Quests | 这是主循环骨架 |
| P0 | Embeddium + ModernFix + FerriteCore + ServerCore | 不先稳性能，后面必崩 |
| P1 | 世界生成与结构 | 没有勘探体验，前期会很空 |
| P1 | 仓储与物流 | 中期自动化手感决定包质量 |
| P1 | 自研物理的“轻量核心” | 先做真实感，不做重模拟 |
| P2 | 自研星际第一版 | 先把 EV-IV 玩通 |
| P2 | 建筑装饰群 | 让工厂真有时代面貌 |
| P3 | 终局戴森工程 | 前中期没稳，终局做再多都白搭 |

## 21. 关键风险

| 风险 | 表现 | 处理策略 |
|---|---|---|
| 科技树被旁系模组跳过 | 玩家不走 GTM 主线 | 关键配方全部 KubeJS 重写 |
| 物理系统过重 | 大基地掉 TPS | 只模拟关键结构，不做全局物理 |
| 航天太复杂导致劝退 | 玩家看不懂上天流程 | 用任务、Ponder、模拟发射训练 |
| 模组太多导致冲突 | 启动崩溃 / 配方错乱 | 先做 50 模最小包，再逐层加 |
| 世界生成过杂 | 资源节奏失控 | 只保留服务主线的群系与结构 |
| 后期仓储性能雪崩 | AE2 / 物流网卡顿 | 限制子网数量，压测样板工厂 |

## 24. 最终建议

如果你要真正把这个整合包做出来，不要一上来就追求“300 模全装”。

正确顺序应该是：

1. 先做 50 模左右的 **主线原型包**
2. 再扩到 120-160 模的 **可玩 alpha**
3. 然后上到 220+ 模的 **完整 beta**
4. 最后再把装饰、世界生成、生活化和长尾库补到 **296 项正式版规划**

这样你会更容易把“工业史递进感”做出来，而不是做成一个内容很多但节奏松散的大杂烩。
